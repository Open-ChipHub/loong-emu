# LoongArch64 CPU Emulator (la-emu) — 架构分析报告

## 1. 项目概述

| 属性 | 详情 |
|---|---|
| 项目名 | loong64-emu |
| 语言 | C (插件使用 C++) |
| 构建系统 | GNU Make |
| 依赖 | glib-2.0, libm, librt |
| 输出产物 | `la_emu_user` (用户态), `la_emu_kernel` (内核态), `la_emu_ref.so` (差分测试) |
| 目标架构 | LoongArch64 (LA464 / Centaur320 / OpenC910) |
| 宿主平台 | x86_64 / LoongArch64 Linux |
| 许可证 | GPL-2.0-or-later |
| 代码规模 | ~25,500 行 C 源码 |
| 性能 | 约为原生执行的 3% |

项目继承 QEMU 代码风格与部分基础设施（CSR 位域宏、TLB 模型、异常框架），但不依赖 QEMU 运行时，是独立轻量级仿真器。

## 4. 执行流程详解

```
程序启动 (main)
  │
  ├─[1] 解析命令行参数 (getopt)
  │     -m 内存大小  -k 内核镜像  -d 日志  -g GDB  -z 确定模式
  │     -w 硬件页表  -s 增强串口   -p 插件  -b BIOS  -c 检查项
  │
  ├─[2] 分配物理内存  ram = mmap(4G)
  │
  ├─[3] 加载镜像
  │     ├── load_elf()         → 内核态 ELF
  │     ├── load_elf_user()    → 用户态 ELF (含 mmap 映射段)
  │     └── load_binary()      → 原始二进制 (BIOS)
  │
  ├─[4] 初始化 CPU
  │     ├── cpu_reset()        → PLV=0, IE=0, DA=1, PG=0
  │     ├── loongarch_core_initfn() → 设置 cpucfg[0..20]
  │     └── cpu_clear_tc()     → 清空 TLB Cache
  │
  ├─[5] 用户态额外步骤
  │     ├── user_setup_stack() → mmap 4G 栈 + 设置 auxv
  │     └── 拷贝 argv/envp 到栈上
  │
  ├─[6] 初始化外设 (内核态)
  │     ├── timer_create()     → 100MHz 定时器 (CLOCK_MONOTONIC)
  │     └── serial_init()     → NS16550 UART (可选增强版)
  │
  ├─[7] 插件加载 (可选)
  │     └── dlopen(plugin.so) → install_func() → plugin_ops
  │
  └─[8] 进入执行
        ├── gdbserver=1  → gdbserver_init(1239) → gdbserver_loop()
        └── gdbserver=0  → exec_env(env)
                              │
                              ▼
  ┌───────────────────────────────────────────────────────────┐
  │               exec_env() 主循环                              │
  │                                                             │
  │  sigsetjmp(env->jmp_env)  ← 异常返回点                        │
  │    │                                                        │
  │    while(1) {                                               │
  │      handle_debug_cli()     ← CLI 模式: 检查调试命令           │
  │      loongarch_cpu_check_irq() ← 检查定时器/串口中断           │
  │      if has_irq → do_interrupt(EXCCODE_INT)  ← 中断向量分发   │
  │      check_gdb_breakpoints()   ← GDB 断点检查                │
  │      check_checkpoint()        ← 检查点触发检查               │
  │      insn = fetch(env, &ic)    ← 取指 (TLB Cache 加速)       │
  │      plugin_emu_insn_before()  ← 插件回调                    │
  │      ret = interpreter(env, insn, ic) ← 译码+执行            │
  │      if !ret → log "illegal instruction"                    │
  │      icount++                                               │
  │      if siglongjmp triggered → do_interrupt(cs)             │
  │    }                                                        │
  └───────────────────────────────────────────────────────────┘
```

## 7. 异常与中断处理

```
┌──────────────────────────────────────────────────────────────────┐
│                        异常触发路径                                 │
│                                                                  │
│  执行异常 (指令触发):                                              │
│    interpreter() → CHECK_FPE / CHECK_PLV → do_raise_exception()  │
│                                                                  │
│  访存异常 (MMU触发):                                               │
│    ram_ld/st → get_physical_address() → raise_mmu_exception()    │
│                                                                  │
│  外部中断 (异步):                                                  │
│    exec_env() → loongarch_cpu_check_irq() → cs->exception_index  │
│                                                                  │
│  所有异常最终汇聚:                                                  │
│    do_raise_exception(env, code, pc)                             │
│      ├── cpu_clear_tc(env)          ← 清空 TLB Cache              │
│      ├── cs->exception_index = code                              │
│      └── siglongjmp(cs->jmp_env, 1) ← 跳出解释循环                │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                     loongarch_cpu_do_interrupt()                  │
│                                                                  │
│  根据 exception_index 分发:                                        │
│                                                                  │
│  ┌─────────────────────────────────────────────┐                 │
│  │ DBP (调试断点):                                │                 │
│  │   CSR_DBG.DCL=1, ECODE=0xC                   │                 │
│  │   CSR_DERA=PC, 跳转 EENTRY+0x480              │                 │
│  ├─────────────────────────────────────────────┤                 │
│  │ INT (中断):                                   │                 │
│  │   如果 DST=1 → 设置 DEI=1, 走 DERA 路径        │                 │
│  │   计算优先级: vector = 31 - clz32(pending)     │                 │
│  │   跳转: EENTRY + (EXCCODE_EXT_INT+vector)*VS  │                 │
│  ├─────────────────────────────────────────────┤                 │
│  │ TLB 重填 (ISTLBR=1):                          │                 │
│  │   保存: TLBRPRMD.PPLV ← PLV, .PIE ← IE        │                 │
│  │   DA=1, PG=0                                  │                 │
│  │   跳转: PC ← TLBRENTRY                        │                 │
│  ├─────────────────────────────────────────────┤                 │
│  │ 普通异常:                                      │                 │
│  │   保存: PRMD.PPLV ← PLV, PRMD.PIE ← IE        │                 │
│  │   ERA ← PC                                    │                 │
│  │   ESTAT.ECODE ← MCODE(cause)                  │                 │
│  │   跳转: PC ← EENTRY + MCODE(cause)*VS         │                 │
│  └─────────────────────────────────────────────┘                 │
│                                                                  │
│  统一操作:  CRMD.PLV=0, CRMD.IE=0  (切换到最高特权级, 关中断)       │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                        异常码一览                                   │
│                                                                  │
│  EXCCODE_INT    (0)  外部中断                                      │
│  EXCCODE_PIL    (1)  页面无效 - 读                                  │
│  EXCCODE_PIS    (2)  页面无效 - 写                                  │
│  EXCCODE_PIF    (3)  页面无效 - 取指                                │
│  EXCCODE_PME    (4)  页面修改异常                                   │
│  EXCCODE_PNR    (5)  页面不可读                                     │
│  EXCCODE_PNX    (6)  页面不可执行                                   │
│  EXCCODE_PPI    (7)  页面特权错误                                   │
│  EXCCODE_ADEF   (8)  地址错 - 取指                                 │
│  EXCCODE_ADEM   (8.1)地址错 - 访存                                 │
│  EXCCODE_SYS    (11) 系统调用                                      │
│  EXCCODE_BRK    (12) 断点指令                                      │
│  EXCCODE_INE    (13) 非法指令                                      │
│  EXCCODE_IPE    (14) 指令特权错误                                   │
│  EXCCODE_FPD    (15) FPU 禁用                                      │
│  EXCCODE_SXD    (16) 128位 SIMD 禁用                               │
│  EXCCODE_ASXD   (17) 256位 SIMD 禁用                               │
│  EXCCODE_FPE    (18) 浮点异常                                      │
│  EXCCODE_BCE    (10) 边界检查异常                                   │
│  EXCCODE_DBP    (26) 调试断点                                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 8. 指令解释器架构 (interpreter.c)

```
┌──────────────────────────────────────────────────────────────────┐
│                     interpreter(env, insn, ic)                     │
│                                                                  │
│  输入:  env  = CPU 状态指针                                        │
│         insn = 32-bit LoongArch 指令                              │
│         ic   = 指令缓存指针 (可为 NULL)                              │
│                                                                  │
│  返回:  true  = 正常执行 (PC += 4)                                 │
│         false = 非法指令                                          │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  执行步骤:                                                         │
│                                                                  │
│  1. 权限检查                                                       │
│     ├── CHECK_FPE(bytes): FPU/LSX/LASX 使能检查                   │
│     ├── CHECK_PLV(plv): 特权级检查 (仅内核态)                       │
│     └── 失败 → do_raise_exception(), return                       │
│                                                                  │
│  2. 缓存查找                                                       │
│     if ic ≠ NULL and ic->trans_func:                             │
│       return ic->trans_func(env, ic)   ← 直接调用缓存函数          │
│                                                                  │
│  3. 指令解码 (decode)                                              │
│     switch(insn bits) {                                          │
│       算术:  add/sub/mul/div/mod/...                             │
│       逻辑:  and/or/nor/xor/...                                  │
│       移位:  sll/srl/sra/rotl/rotr/...                          │
│       分支:  beq/bne/blt/bge/bltu/bgeu/beqz/...                 │
│       跳转:  jirl/b/bl                                           │
│       访存:  ld.b/h/w/d, st.b/h/w/d                              │
│              ldx.b/h/w/d, stx.b/h/w/d                           │
│              ldptr, stptr (PTE 遍历访存)                          │
│       CSR:   csrrd/csrwr/csrxchg                                │
│       TLB:   tlbsrch/tlbrd/tlbwr/tlbfill/tlbclr/tlbflush       │
│              invtlb_all/invtlb_all_g/invtlb_*                    │
│       原子:  ll.w/sc.w                                            │
│       特权:  ertn/idle                                            │
│       杂项:  rdtime/dbar/ibar/cacop/cpucfg/crc/...              │
│     }                                                            │
│                                                                  │
│  4. 解码结果缓存 (cpu_put_ic)                                      │
│     将 trans_func 和 args 存入 INSCache[PC & 0x3FFF]             │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                     指令缓存 (INSCache)                            │
│                                                                  │
│  IC_NUM = 1 << 14 = 16384 条目                                    │
│  IC_INDEX(va) = (va >> 2) & 0x3FFF                               │
│                                                                  │
│  每个条目:                                                         │
│  ┌────────────┬────────┬────────┬────────┬────────┬────────┐     │
│  │ trans_func │ arg[0] │ arg[1] │ arg[2] │ arg[3] │  insn  │     │
│  └────────────┴────────┴────────┴────────┴────────┴────────┘     │
│                                                                  │
│  命中率: ic_hit_count 统计                                        │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 16. 总结

本项目是一个功能完备的 **LoongArch64 全系统仿真器**，具备以下核心能力：

- **完整的 LA64 指令集模拟**：覆盖基础整数、浮点、LSX/LASX SIMD 扩展，可运行 Linux 6.10 内核和 SPEC CPU 基准测试
- **双重运行模式**：用户态（低开销直接映射）和内核态（完整 MMU/TLB/外设模拟）
- **丰富的调试设施**：GDB 远程调试、内建 CLI、差分测试、检查点、性能计数器、插件系统
- **设计继承 QEMU** 的成熟模式（CSR 位域宏、TLB 结构、异常框架），保持代码可读性和可维护性
- **执行性能**：通过 INSCache 和 TLB Cache 两级缓存优化，可达原生 3%

**适用场景**：OS 内核开发调试、固件验证、CPU RTL 差分测试、SPEC CPU 性能分析、LoongArch 学习研究。
